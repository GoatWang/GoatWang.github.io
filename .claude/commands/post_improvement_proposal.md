Review the blog post at $ARGUMENTS (path to the markdown file in content/posts/).

## Instructions

1. Read the full post content
2. Optionally view the rendered version on the local Hugo dev server if running
3. Write a detailed improvement proposal to the `prompts/` folder (follow CLAUDE.md naming convention) covering:

### Title
- Propose 3-5 alternative titles ranging from neutral to provocative
- Consider click-through appeal, accuracy, and tone

### Structure & Paragraph Arrangement
- Analyze the current section order and identify narrative flow issues
- Propose a reordered structure with rationale (e.g., move context earlier for tension, merge sections for pacing)
- Identify sections that feel redundant or can be consolidated

### Tone & Style
- Identify passages that are too neutral, too hedging, or too textbook-like
- Propose specific rewrites with more attitude: sharper openings, ironic observations, absurd contrasts, punchier conclusions
- Suggest where to cut unnecessary qualifiers and hedging language

### Detail Fixes
- Typos, duplicate words, broken formatting
- Redundant content (e.g., translating a quote then restating the same idea)
- Image/link issues

### Summary Table
- Prioritize all suggestions (high/medium/low) with expected impact

## Output
Write the proposal as a markdown file in the `prompts/` folder and present a summary to the user.
